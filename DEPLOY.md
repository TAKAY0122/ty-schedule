# RB事業2課 スケジュール管理アプリ — 公開手順書

## 構成

- **Cloudflare Workers**(無料枠で十分動作します)
- **Cloudflare D1**(データベース・無料枠)
- 全部で1つのWorkerにまとまっています。`wrangler deploy` 一発で公開されます。

```
ty-schedule/
├ wrangler.toml      … Cloudflareの設定(★D1のIDを貼る場所あり)
├ schema.sql         … データベースの初期化SQL(新規作成用・時給テーブル込み)
├ migrate-wage.sql   … 既存DBに給与計算を追加する更新用SQL(更新時のみ)
├ package.json
├ gas-連携.gs        … スプレッドシート側に貼る連携スクリプト(任意)
├ DEPLOY.md          … この手順書
├ src/index.js       … サーバー(API・給与計算・21時の催促通知)
└ public/            … 画面(HTML/CSS/JS)
```

## 公開手順A(おすすめ:PCにNode.jsがある場合)

ターミナルでこのフォルダに移動して、以下を順番に実行してください。

```bash
# 1. 準備
npm install
npx wrangler login        # ブラウザが開くのでCloudflareアカウントで許可

# 2. データベースを作成
npx wrangler d1 create schedule-db
```

→ 実行結果に `database_id = "xxxxxxxx-xxxx-...."` と表示されます。
→ **その値を `wrangler.toml` の「ここにD1データベースのIDを貼り付けてください」の部分に貼り付けて保存**してください。

```bash
# 3. データベースの中身(テーブル)を作成
npx wrangler d1 execute schedule-db --remote --file=schema.sql

# 4. 公開
npx wrangler deploy
```

→ `https://ty-schedule.○○○.workers.dev` というURLが表示されたら完了です。

> **【既に稼働中のアプリを更新する場合のみ】給与計算の導入**
> すでにデプロイ済みのアプリに今回の給与計算機能を追加するときは、**1回だけ**マイグレーションを実行してください(スケジュール表に業務名などの列・時給テーブルを追加します)。
> ```bash
> npx wrangler d1 execute schedule-db --remote --file=migrate-wage.sql
> ```
> その後 `npx wrangler deploy` で最新コードを公開します。
> ※ 新規に `schema.sql` で作成する場合は、最初から入っているので **不要** です。

## 公開手順B(GitHub連携で公開する場合)

1. GitHubに新しいリポジトリを作り、このフォルダの中身をアップロード(`node_modules` は不要)
2. Cloudflareダッシュボード → **ストレージとデータベース → D1** → 「データベースを作成」→ 名前 `schedule-db`
3. 作成したD1を開き、**コンソール**タブで `schema.sql` の中身を全部貼り付けて実行
4. D1の概要画面に表示される **データベースID** をコピーし、`wrangler.toml` の該当箇所に貼り付けてGitHubにプッシュ
5. Cloudflareダッシュボード → **Workers & Pages → 作成 → Workers → リポジトリをインポート** で該当リポジトリを選択
   - ビルドコマンド:空欄のまま / デプロイコマンド:`npx wrangler deploy`(初期値のまま)
6. デプロイ完了後に表示されるURLにアクセス

以降はGitHubにプッシュするたびに自動で再デプロイされます。

| 初期パスワード | **323331**(登録番号と同じ) |
| 手配者専用パスワード(PIN) | **111111**(初期値・管理者画面で変更可能) |

1. URLを開き **323331 / 323331** でログイン
2. 右上メニュー(☰)→「パスワード変更」で管理者パスワードを変更
3. 「アカウント管理」からメンバーを登録(初期パスワードは各自の登録番号)

## 各機能の場所

- **スケジュール入力**:右上メニュー →「手配者専用ページ」→ PIN入力 → 上部に「スケジュール入力」が出現(手配チーム以上のみ)
- **編集履歴・ログイン中メンバー**:手配者専用ページ内
- **新人報告**:全員提出可。メンツは1次まで、チーフは2次まで入力。提出するとチーフ全員にベル通知
- **2次チェック**:「報告一覧」→ 該当の報告をタップ → 2次入力(チーフ以上)
- **ドラフトリスト**:2次チェックで「承認OK」になった候補者が自動で並びます
- **🔰新人通知**:報告の「次回現場名・日付」がスケジュールと一致すると、その現場に入るメンバーへ通知+表に🔰バッジ
- **催促通知**:毎日21時に、その日現場に入っているチーフで報告未提出の人へ自動通知
- **会場タップ**→Googleマップ / **現場名タップ**→同じ現場のメンバー一覧・備考
- **ブラックリスト**:提出・閲覧ともチーフ以上。日付・報告者・名前・会話/服装/身なり/遅刻/業務(各5段階)・理由の9項目
- **育成計画**:スケジュール表の「育成計画」列をタップして編集(チーフ以上)。本人スケジュールにのみ表示
- **時間・給与・IN/OUTの閲覧制限**:これらの列は手配担当(handler)以上のみ表示。チーフ・メンツは現場名・会場までしか見えません(本人でも同じ)
- **1日複数現場**:同じ人が同じ日に複数の現場に入る場合、スケジュール表では1日の行内に重ねて表示されます。スケジュール入力では「＋現場」で行を追加できます
- **現場一覧**(チーフ以上):月ごとに、登録済みの現場が日付順で一覧表示されます。現場をタップするとその現場のメンバーが出ます(マイスケジュールの現場タップと同じ表示)。手配者モード中は、メンバー表示の下から「＋この現場にメンバーを追加」で、複数人・人ごとに複数日を指定して追加できます。
- **複数人に一括登録**(手配者モード):スケジュール入力ページの上部。現場名・会場・時間を共通で入れ、メンバーを追加してから、メンバーごとに日付と備考を個別指定して一度に登録できます(例:Aさんは6/1・6/5 / Bさんは6/1・6/8)。
- **メニュー**:画面の移動は右上(スマホは左上)の三本線(☰)メニューから行います。役割に応じて表示される項目が変わります
- **担当手配者**:アカウント管理で各メンバーに「担当手配者」を設定できます。設定すると、スケジュール閲覧の「他のメンバーを見る」やスケジュール入力で、担当手配者ごとにメンバーを絞り込んで選べます(担当手配者プルダウン→メンバープルダウンの2段)
- **入力者名の自動付与**:備考と育成計画は、入力した人の姓が自動で末尾に付きます(例:「物販頭」→「物販頭(吉崎)」)。後から別の人が編集すると、その人の名前に置き換わります
- **新人(🔰)表示**:マイスケジュールでの新人バッジ表示はチーフ以上のみ。メンツには表示されません
- **メンツへの機能秘匿**:メンツには、チーフ以上だけが使う機能(報告一覧・2次チェック・ブラックリスト・ドラフト・スケジュール入力・取り込み・アカウント管理・手配者専用ページ)を、メニューにもページにも一切見せません。直接URLを開いても「ページが見つかりません」とだけ表示され、機能の存在は分かりません。サーバー側もメンツには該当APIを返さず、新人報告フォームの2次項目(やる気・表情・総合点・ドラフト・育成計画)もメンツには表示されません

## スプレッドシートURL取り込み(手配担当以上・毎日違うシート向け)

毎日違うスプレッドシートのURLを貼るだけで、IN/OUT・現場・会場をアプリに取り込めます。手配者専用ページの「スプレッドシートから取り込み」で使います。

**使い方**
1. 取り込みたいシートを開き、「共有」→「リンクを知る全員」を**閲覧可**にする
2. そのURLをコピーし、手配者専用ページの入力欄に貼る(複数URLは改行で区切って同時に取り込み可)
3. 対象月を選び(個人スケジュール表の場合に使用)、必要なら「フォーマット」を指定(通常は自動判定でOK)
4. 「取り込み実行」を押すと、各URLごとに反映件数が表示されます

**対応フォーマット(3種類・自動判定)**
- **IN/OUT表(勤務表)**: 1日1現場分の勤務者リスト。登録番号・業務名(現場名)・開始/退勤時間・日付・会場を読み取ります。**IN/OUT時刻はここから取り込まれます**
- **個人スケジュール表(月間表)A・B**: メンバーが横方向に3列セット(現場名・会場・備考)で並ぶ月間表。現場名・会場を読み取ります

**ポイント**
- 取り込んだ内容は**その時点でアプリのDBに保存**されます。後でシートを非公開に戻したり削除したりしても、アプリ内のデータは残ります(「URL入力時だけ公開、後で非公開」という運用に対応)。
- アプリが直接シートを読むため、シートは取り込みの瞬間に「リンクを知る全員が閲覧可」である必要があります(非公開シートは読めません)。
- 1日に複数現場がある人も、複数行として取り込まれます。
- 「既存に追加」にチェックすると、その日の既存の現場を残したまま追加します(通常はオフ=その日を置き換え)。
- 取り込み分は編集履歴に「スプレッドシートURL」と記録されます。
- ※ 列の位置は標準フォーマットに合わせています。実際のシートで列がずれて読めない場合は、列位置の調整が必要なのでご連絡ください。

## スプレッドシート連携(GAS・シート側から自動送信)

URL取り込みとは別に、シート側にスクリプトを仕込んで「編集したら自動でアプリへ送る」方式も使えます。同梱の `gas-連携.gs` を使います。

**設定手順**
1. アプリの「アカウント管理 → スプレッドシート連携」で**取り込みトークン**をコピー
2. スケジュールを入力するGoogleスプレッドシートを開き、「拡張機能 → Apps Script」を開く
3. `gas-連携.gs` の中身を全部貼り付け、先頭の `CONFIG` の `APP_URL`(アプリのURL)と `IMPORT_TOKEN`(コピーしたトークン)を書き換えて保存
4. 関数 `setup` を一度だけ実行(初回は権限の承認を求められます)

これで、シートを編集して保存するたびに内容がアプリへ自動反映されます。手動で送りたいときはシートのメニュー「アプリ連携 → 今すぐ全部送信」を使います。

**シートの列(1行目はヘッダー、2行目以降にデータ)**
A:登録番号 / B:日付(2026-06-15) / C:種別(現場・休暇・有給・×) / D:現場名 / E:会場 / F:IN(10:15) / G:OUT(19:45) / H:給与(空欄なら自動計算) / I:備考

- 登録番号と日付が空の行はスキップされます。未登録の登録番号も無視されます(送信後に件数と注意が表示されます)。
- **アプリ内で直接編集した分はアプリにのみ保存**され、シートには戻りません。**シートを編集した分はGAS経由でアプリに反映**されます。スプレッドシート取り込み分は、手配者専用ページの編集履歴に「スプレッドシート」と記録されます。
- トークンが漏れた場合などは、アカウント管理画面で**再発行**できます(再発行後はGAS側のトークン貼り替えが必要です)。

## 給与計算(RB事業2課ルール)

業務名ごとに、ランク×時期の時給で計算します。スケジュールの日別編集で **業務名・搬入終了・終演・2st** を入力すると、給与が自動計算されます。

**時給**:ランク(A〜E)×時期(〜2025/9 旧 / 2025/10〜 改定)×種別(案内料金 / 搬入出料金)。
→ 管理画面の「💴 時給設定」でいつでも編集できます(管理者のみ)。

**業務名の区分**
- 案内料金:案内・受付・案内・準備・本部付・制作補助・運営補助・雑務
- 搬入出料金:準備・設営・搬入・搬出・機材搬入・機材搬出・ステージハンド
- 時間帯で分割:搬入・案内(IN〜搬入終了=搬入、以降=案内)/ 案内・搬出(IN〜終演=案内、以降=搬出)/ パッケージ(搬入→案内→搬出)
- 対象外:ケータリング・物品販売、および A〜E 以外のランク

**手当・時間**
- 深夜手当(22:00以降)= 深夜時間 × 案内料金 × 0.25(加算)
- 超過手当(13時間超)= 超過時間 × 案内料金 × 0.25(加算)。深夜と重複しても各0.25が独立で乗ります
- 2st(複数回公演)= +¥500 / 休憩控除なし(IN〜OUTで計算)
- IN=開始時間、OUT=退勤時間(取り込み時は「退勤時間」と「終了予定時間」の遅い方を採用)
- スケジュール表の「時間外(残業)」列は **9時間を超えた分** を表示します(表示用)。給与の超過手当は **13時間超** で計算します(別物)

**給与確定(ロック)**:現場日から **2週間** を過ぎた日は編集できません(保存・一括登録・現場一括編集すべてでブロック)。時給を後から変更しても、確定済みの日は再計算されません(当時の時給で固定)。

**過去データの修正**:アカウント管理 →「💴 時給設定」→「過去データの給与・残業を再計算」で、取り込み済みの全現場を現在の時給・新ルールで計算し直せます(残業9h・業務名込み。手動入力の給与も上書き)。

**表示**:給与・時間・IN/OUT・**業務名** は手配担当(handler)以上のみ表示。スケジュール表に「業務」列が出ます。「給与(手動)」に金額を入れると、その値が自動計算より優先されます(有給の金額入力などに使用)。

## スマホ・PC・タブレット対応

このアプリはスマホ・PC・タブレットのどのブラウザでも使えるレスポンシブ対応です。画面の幅に合わせてレイアウトが自動で変わり、スマホではメニューやボタンがタップしやすいサイズになります。スケジュールや入力の表は、画面に収まらない列(IN/OUT・給与など)は横スクロールで確認できます。インストールは不要で、URLを開くだけで使えます。

## 動くデモ(preview.html)

`preview.html` はサーバー不要で動く操作デモです。ダブルクリックでブラウザで開くだけで、ログインから現場入力・新人報告・各画面の確認まで実際に操作して使用感を試せます(データはブラウザのメモリ内のみで、閉じると消えます)。上部のデモバーで「メンツ/チーフ/手配/管理者」を切り替えてログインでき、手配・管理者は自動で手配者モードに入るのですぐ現場入力を試せます。デモ用のログイン番号:管理者=371166 / 手配=333333 / チーフ=323331 / メンツ=322222(パスワードは番号と同じ)。
